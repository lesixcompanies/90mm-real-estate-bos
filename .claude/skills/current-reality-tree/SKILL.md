# Current Reality Tree (CRT)

## Purpose
Map the cause-and-effect chain from visible symptoms to root cause.
Identify the ONE constraint that, if addressed, would have the greatest
positive impact on the entire system.

## How This System Uses It
The diagnostic IS a conversational CRT. The opening question and
five rounds of follow-up questions trace the causal chain from "tell me
about the people you serve" down to the root cause.

## When to Re-Run
- The current constraint has been resolved and a new one is emerging
- The agent describes symptoms that don't connect to the known constraint
- Quarterly review suggests the business reality has shifted significantly

## Process
1. Start with observable symptoms (what the agent experiences daily)
2. Ask "why" to trace each symptom back to its cause
3. Look for where multiple symptoms converge on a single cause
4. That convergence point is likely the constraint
5. Validate by checking: if this one thing changed, would the symptoms improve?

## Output
Update context/constraints.md with:
- Stated constraint (agent's words)
- Observed constraint (what the analysis reveals)
- Constraint category
- Root cause
- Through-line

## Resistance Layers Addressed
- Layer 0: Does the agent agree a problem exists?
- Layer 1: Do we agree on what the problem is?
- Layer 2: Does the agent believe it's within their control?

All three must be resolved before advancing to the Evaporating Cloud.

## What Happens Next
After the CRT is complete and Layers 0-2 are resolved → move to
Evaporating Cloud. Do NOT jump to solutions or execution.
